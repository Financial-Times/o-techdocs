Sample content (h1)

The following is sample content demonstrating all the built in styles supported by the tech docs module. We use standard FT styles where possible, and basics such as headings (H1 to H5), bulleted and numbered lists, inline links and <code> are obviously supported, but the style samples here are designed to better support technical use cases rather than editorial ones.

The code examples are in the source of this page.

Tables

o-table is used for table styling. At a minimum, give tables a class of o-table, but see o-table's documentation for more detail.

To make tables horizontally scrollable on small screens, wrap them in a div, ie <div class="o-techdocs-table"><table>...</table></div> or include the techdocs JavaScript module, which will do that for you.

Code

For inline code, wrap the content in a <code> tag.

For blocks, wrap in <pre><code>:

$.ajax({
    url: "/api/getWeather",
    data: {
    zipcode: 97201
    },
    success: function( data ) {
    $( "#weather-temp" ).html( "" + data + " degrees" );
    }
    });

To trigger syntax highlighting using prettify, prefix the <pre> with a prettify inline hint such as <?prettify linenums=1?>, or add the prettify class, as documented in the prettify wifi. Since prettify auto-loads all its lang files, we use it from the Cloudflare CDN. Put this at the end of your <body> to enable support:

<script src='//cdnjs.cloudflare.com/ajax/libs/prettify/r298/prettify.js'></script>
    <script>prettyPrint()</script>

Images

Images will be auto-scaled to the width of the content area and given a border.

Asides

Use the <aside> tag to create inset boxed content, useful for examples, non-normative notes, warnings for the wary etc. For a heading inside the aside, use an <h5>.

Example heading for the aside

Where features or elements of design are used in multiple websites, we need to build them to be usable in all those cases, and portable between them, rather than always building features solely in the context of one product. These components should not be seen as an intrinsic part of any product, but instead as part of the new digital FT.

Disabling doc styles

To make it easier to author content in markdown, styles are applied to naked tags inside any .o-techdocs-content element. However, if you want to include content that you don't want to affect with tech doc styles, you can simply close the .o-techdocs-content and open a new one when you want back into auto-formatting.
