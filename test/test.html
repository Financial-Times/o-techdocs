Sample content

The following is sample content demonstrating all the built in styles supported by the tech docs module. We use standard FT styles where possible, and basics such as headings (H1 to H4), bulleted and numbered lists, inline links and <code> are obviously supported, but the style samples here are designed to better support technical use cases rather than editorial ones.

The code examples are in the source of this page.

Headings

For best results and to enable the nav to auto-index your content easily, use H1 once at the top of the document and then correctly nest any uses of H2 to H4.

To trigger syntax highlighting using prettify, prefix the <pre> with a prettify inline hint such as <?prettify linenums=1?> as documented in the prettify wifi. Syntax highlighting depends on the Google CDN, to avoid including all language files into the standard CSS bundle, so it will only be loaded where required.

Images

Images will be auto-scaled to the width of the content area and given a border.

Asides

Use the <aside> tag to create inset boxed content, useful for examples, non-normative notes, warnings for the wary etc. Fora heading inside the aside, use an <h4>.

Example heading for the aside

Where features or elements of design are used in multiple websites, we need to build them to be usable in all those cases, and portable between them, rather than always building features solely in the context of one product. These components should not be seen as an intrinsic part of any product, but instead as part of the new digital FT.

Disabling doc styles

To make it easier to author content in markdown, styles are applied to naked tags inside any .o-techdocs-content element. However, if you want to include content that you don't want to affect with tech doc styles, you can simply close the .o-techdocs-content and open a new one when you want back into auto-formatting.
